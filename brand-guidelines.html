Brand Guidelines

A living reference for designers and developers. Everything needed to build consistent, on-brand experiences for Robuxindo Store.

01

Brand Identity

Robuxindo Store is Indonesia's trusted Roblox top-up destination. The brand balances gamer-friendly energy with trustworthy professionalism — warm, modern, clean, and approachable. Never cold. Never corporate. Never neon-heavy.

Brand Personality

🚀 Fast 🛡️ Trusted 🎮 Gamer-Friendly ✨ Modern 😊 Playful 💎 Premium-Casual

Logo Usage

ROBUXINDO STORE
Light background

Clear space: Always maintain at least 1× the logo height as clear space on all sides.
Minimum size: Never render the logo smaller than 24px height.
Don't: Stretch, rotate, recolor, or add effects to the logo. Use provided asset files only.
02

Voice & Tone

Robuxindo speaks like a trusted gamer friend — not a stiff corporation. Casual but credible. Energetic but not chaotic. Always honest, always helpful.

Writing Principles

✅ Do
  • → Use informal but professional Indonesian
  • → Be direct and concise
  • → Highlight speed, safety, and value
  • → Use active voice
  • → Speak to gamers as peers
  • → Keep sentences short
❌ Don't
  • → Sound overly formal or corporate
  • → Use unnecessary jargon
  • → Make promises you can't keep
  • → Use ALL CAPS for emphasis
  • → Write walls of text
  • → Sound desperate or pushy

Copy Examples

CTA Button
✅ "Beli Sekarang" / "Top Up Robux" / "Mulai Sekarang"
❌ "Klik Di Sini Untuk Melakukan Pembelian Produk"

Keep CTAs short, action-oriented, and energetic.

Error Message
✅ "Oops! Transaksi gagal. Coba lagi ya."
❌ "ERROR: Transaction processing failed. Please try again."

Be human, empathetic, and guide users to a solution.

Empty State
✅ "Belum ada pesanan. Yuk beli Robux pertamamu!"
❌ "No records found."

Turn empty states into opportunities to guide and delight.

Do's & Don'ts

Quick reference for the most important guardrails when building Robuxindo UIs.

Visual Style

✅ Do
  • → Use warm beige/cream as page backgrounds
  • → Use brand brown for primary CTAs and headings
  • → Use rounded corners (≥12px) on all cards and buttons
  • → Keep shadows soft and warm-toned
  • → Use orange/peach as promotional accents
  • → Keep sections airy with generous spacing
  • → Use green for "aman", verified, and success states
  • → Maintain 4-pt spacing grid throughout
❌ Don't
  • → Use dark mode as the primary/default mode
  • → Use neon colors or cyber/gaming aesthetics
  • → Use pure black (#000000) as a dominant color
  • → Mix more than 3 accent colors in one section
  • → Use sharp 0px corners on cards or primary buttons
  • → Crowd sections — maintain breathing room
  • → Use blue as a primary brand color
  • → Create heavy, layered gradients

Component Usage

✅ Do
  • → One primary button per view/section
  • → Always show visible labels above inputs
  • → Use badge-dot for status indicators
  • → Add hover transitions (150ms ease)
  • → Show meaningful empty states
  • → Provide error messages at field level
❌ Don't
  • → Stack multiple primary CTAs together
  • → Use placeholder text as the only label
  • → Use bare color for status — always pair with dot/icon
  • → Trigger animations over 300ms on interactions
  • → Show generic "No data" empty states
  • → Show only a toast for form validation errors

Logo & Brand

✅ Do
  • → Use provided logo asset files as-is
  • → Maintain clear space around the logo
  • → Place logo on light or brand-dark backgrounds
  • → Keep logo minimum 24px height
❌ Don't
  • → Stretch or distort the logo
  • → Recolor or add effects to the logo
  • → Place logo on busy photo backgrounds
  • → Create new logo variants without approval
